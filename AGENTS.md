\# SN POS V2



Small clothing store sales management web app.



One user only.



Do not over-engineer.



\## Product Goal



Clean, beautiful, fast, easy to use.



Pink-first UI.



Desktop-first, but must work well on tablet and mobile.



\## Core Modules



\- Dashboard

\- POS / Sales

\- Products

\- Inventory

\- Orders

\- Customers

\- Reports

\- Settings



No employee management.

No roles/permissions.

No multi-branch system.

No CRM/marketing features.

No enterprise bloat.



\## Design Direction



Modern premium retail software.



Not a generic admin template.



Dark mode by default.



Pink is the main accent.



Use soft surfaces, clean borders, good typography, compact spacing.



Avoid huge cards, oversized buttons, weak contrast, messy layouts, and too much empty space.



\## UX Rules



Fast cashier workflow.



Important info visible at a glance.



Few clicks.



Responsive layout:

\- Desktop: full experience

\- Tablet: optimized layout

\- Mobile: no broken UI, simplified layout



\## Code Rules



React + TypeScript + Vite.



Tailwind CSS v4.



shadcn/ui.



Use reusable components.



Keep code simple.



Before building UI, design first.
